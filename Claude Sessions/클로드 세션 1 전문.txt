`packageProductDependencies` (타겟 프레임워크)는 반영됐지만, 프로젝트 레벨 Package Dependencies 탭은 **Xcode가 직접 패키지를 fetch하고 Package.resolved를 생성해야 표시**되는 구조라 UI로 추가하는 게 확실해.

---

직접 추가하는 방법이야:

1. Xcode에서 `.xcworkspace` 열기
2. 좌측 파일 트리 상단의 **프로젝트 파일** 클릭
3. **Package Dependencies** 탭 선택
4. **`+`** 버튼 클릭 후 아래 URL 순서대로 추가

```
https://github.com/uber/RIBs
https://github.com/SnapKit/SnapKit
https://github.com/Alamofire/Alamofire
```

5. 각각 **Up to Next Major Version** 선택 후 추가

패키지 추가 후 Xcode가 자동으로 fetch하고 `Package.resolved`까지 생성해 줄 거야. 이미 `packageProductDependencies`는 등록되어 있으니 타겟 연결은 자동으로 잡힐 거야.

## 코드 스타일

**Extension 순서**
```
클래스/구조체 본문
// MARK: - Private
// MARK: - Const   ← 항상 가장 아래
```

**MARK와 extension 사이 공백 없음**
```swift
// MARK: - Private
extension Foo {
```

**Extension 분리 원칙**
- Routing 메서드 → `// MARK: - [RIBName]Routing` extension
- ViewControllable 메서드 → `// MARK: - [RIBName]ViewControllable` extension
- Protocol 준수 → 각각 별도 extension

**상수 관리**
- `identifier`, `key`, `limit` 등 모든 상수는 `enum Const` extension으로 관리
```swift
// MARK: - Const
extension Foo {
    enum Const {
        static let identifier = "Foo"
    }
}
```

**이벤트 전달 방식**
- Closure 대신 Delegate Protocol 방식 사용
- 메서드 네이밍: `func deleteKeywordTapped(on cell:)` 형태

## RIBs 아키텍처

**Lifecycle**
- Viewable RIB은 `didBecomeActive` 대신 `viewDidLoad` 사용

**역할 분리**
- Snapshot 생성은 Interactor의 역할, ViewController는 `applySnapshot(_:)` 호출만
- 각 RIB의 비즈니스 로직은 해당 RIB에서만 관리 (e.g. 검색 기능은 SearchResult RIB 전담)

**Listener 네이밍**
- 여러 RIB이 공유하는 ViewController는 listener를 명확히 구분
```swift
weak var searchListener: SearchPresentableListener?
weak var searchResultListener: SearchResultPresentableListener?
```

**Repository**
- `actor` 기반으로 thread-safe하게 구현
- 상수는 `enum Const` extension으로 분리

---

## 폴더 구조

- 여러 RIB에서 공용으로 사용되는 컴포넌트는 `Common` 폴더에 위치
